Clash 的介绍、安装、使用教程

一篇文章讲清楚clash.

介绍

“Clash” 是一个多协议代理工具，它允许您在计算机或移动设备上配置和管理多个网络代理协议，以实现网络流量的隐私保护和访问控制。它支持多种代理协议，例如 Shadowsocks、VMess、Trojan、Socks5 等，使您能够自由选择适合您需求的代理方式。

Android

版本区别

  1. foss和premuium有什么区别？
    foss表示的是开源版本，
    premuium是闭源版本，
    foss和Premuim版本的clash可以同时在同一设备上安装，打开两个应用的关于界面可以看到他们之间的差异
  2. arm64-v8a、armeabi-v7a、x86、x86_64、universal各个版本之间，有什么区别？

  • arm64-v8a：第8代、64位ARM处理器。目前，国内Android应用生态正在向64位架构过渡。同时，移动芯片平台也将逐步弱化对32位应用的支持。

  • armeabiv-v7a：第7代及以上的 ARM 处理器，是目前主流版本，大部分Android设备都使用它。

  • x86 / x86_64：x86 架构的手机都会包含由 Intel 提供的称为 Houdini 的指令集动态转码工具，实现对arm .so 的兼容。考虑 x86不到1% 的市场占有率，x86 相关的两个 .so 也是可以忽略的。

  • universal：通用版本，但是体积较大，如果你不知道如何选择，选择这个是明智的。

有时候会产生奇怪的需求

  • Linux服务器需要完整的网络环境
  • 对外提供HTTP、SOCKS的完整网络代理服务

  1. 进入用户目录，下载，解压，重命名，添加执行权限

  2. 第一次启动

    Clash运行时需要Country.mmdb，当第一次启动 Clash 时会自动下载（会下载至 ~/.config/clash 文件夹下）。自动下载可能会因网络原因较慢，使用访问链接手动下载

使用

Linux

  1. 需要订阅链接下载配置文件

配置 systemd 服务

Linux 系统使用 systemd 作为启动服务器管理机制，首先把 Clash 可执行文件拷贝到 /usr/local/bin 目录，相关配置拷贝到 /etc/clash 目录。

  1. 创建文件夹，移动程序和配置文件

  2. 创建 systemd 服务配置文件/etc/systemd/system/clash.service

  3. 开机启动

  4. 启动clash

为代理添加密码(可选，如果是服务器，不加密码容易永久封禁)

修改配置文件config.yaml

  • http
  • socks

记得修改配置为允许局域网连接，模式修改为rule
